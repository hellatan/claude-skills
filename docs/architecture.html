Known failure modes & fixes

SymptomRoot causeFixStatus
Merges to main never tag a release; release-please opened its PR against develop release-please defaults to the repo's default branch — which is develop under gitflow Pin target-branch: main in release-please.yml (#44) fixed
Release & develop→main promo PRs stuck in action_required; CI never goes green on its own GITHUB_TOKEN-authored bot PRs get no CI runs Author them with the RELEASE_PLEASE_TOKEN PAT (#46) fixed
A scaffolded repo's branch protection waits forever on a check that never reports Hard-coded status-check contexts drift from the real CI job names — renaming a CI job/check is a breaking change for consumer repos gitflow-init derives contexts from ci.yml (#16); treat check renames as breaking fixed
First release on a freshly scaffolded repo never auto-tags release-please config/manifest mismatch (wrong component, 1.0.0 bootstrap) Aligned configs with the verified single-package template (#22, #23, #24, #37) fixed
Every merge to develop runs CI twice Scaffolded workflows fired on both pull_request and push to develop Dropped the duplicate push-on-develop trigger (#63) fixed
Skill edits don't show up when running as the ht-skills plugin The plugin loader caches SKILL.md content at session start Develop via the symlink install (scripts/install.sh) or /reload-plugins fixed

Key files & paths

  • skills/<skill-name>/SKILL.md — one skill per folder; frontmatter name must match the folder
  • skills/<skill-name>/references/ — templates & verified configs the skills write into target repos
  • scripts/validate.sh — SKILL.md sanity checks (the same script CI runs)
  • scripts/install.sh — symlinks every skill into the user's Claude skills directory
  • .github/workflows/validate.yml — CI: runs validate.sh on PRs & pushes
  • .github/workflows/release-please.yml + develop-to-main-pr.yml — the release train
  • .claude-plugin/plugin.json — ht-skills plugin metadata
  • docs/architecture.html — this file (keep it current)
Operate it: ./scripts/validate.sh before every PR · ./scripts/install.sh to symlink skills for live development · comment /rebuild on a PR to re-run CI. To release: merge the auto-opened develop → main PR, then merge release-please's versioned release PR (that merge tags).
