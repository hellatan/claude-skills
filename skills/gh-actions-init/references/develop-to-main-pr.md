# develop → main auto-PR

Companion to `release-please.md`. Closes the one manual gap in the gitflow → release pipeline: nothing otherwise opens the `develop` → `main` PR, so every release starts with the user remembering to open it by hand.

This workflow auto-opens (and keeps refreshed) a **draft** PR from `develop` to `main` whenever `develop` advances. Merging that PR is still a deliberate human action — it's the release trigger. release-please then takes over and opens its versioned release PR against `main`.

## When to scaffold this

- Only for projects with a `develop` branch (gitflow). `main`-only projects don't need it.
- Only when **staging is NOT opted in**. For projects with a `stage` branch the topology is `develop → stage → main`, which needs two coordinated workflows — out of scope here; scaffold the stub and leave a note.

## Prerequisite: Actions must be allowed to open PRs

This workflow calls `gh pr create`, which fails with `GraphQL: GitHub Actions is not permitted to create or approve pull requests` unless the repo setting **Settings → Actions → General → Workflow permissions → "Allow GitHub Actions to create and approve pull requests"** is enabled. `project-scaffold` enables this automatically on freshly created repos (see `project-scaffold/references/step-17-create-repo-push.md`). For an existing repo, enable it once:

```bash
gh api -X PUT repos/{owner}/{repo}/actions/permissions/workflow \
  -F default_workflow_permissions=write \
  -F can_approve_pull_request_reviews=true
```

## Workflow

`.github/workflows/develop-to-main-pr.yml`:

```yaml
name: develop → main PR

# Auto-opens (or refreshes) a draft PR from `develop` to `main` whenever
# `develop` advances. Merge it when ready to release — release-please then
# takes over and opens its versioned release PR.
#
# Rationale: release cadence stays a deliberate human decision (merging this
# PR is the trigger), but you never have to remember to open the develop→main
# PR yourself.

on:
  push:
    branches: [develop]

permissions:
  pull-requests: write
  contents: read

concurrency:
  group: develop-to-main-pr
  cancel-in-progress: false

jobs:
  sync-pr:
    name: open or refresh develop → main PR
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v6
        with:
          # Full history so we can diff develop vs main.
          fetch-depth: 0

      - name: Count commits ahead of main
        id: check
        run: |
          ahead_count=$(git rev-list --count origin/main..origin/develop)
          echo "ahead=$ahead_count" >> "$GITHUB_OUTPUT"
          echo "develop is $ahead_count commit(s) ahead of main."

      - name: Build PR body
        if: steps.check.outputs.ahead != '0'
        run: |
          {
            echo "## Pending release"
            echo ""
            echo "This PR is auto-opened and auto-refreshed whenever \`develop\` advances."
            echo "Merge it when you're ready to release the accumulated features —"
            echo "[release-please](../actions/workflows/release-please.yml) will then"
            echo "open a versioned release PR against \`main\` with a CHANGELOG and"
            echo "version bump."
            echo ""
            echo "### Commits ahead of \`main\` (${{ steps.check.outputs.ahead }})"
            echo ""
            git log --pretty=format:'- %s (`%h`)' origin/main..origin/develop
            echo ""
            echo ""
            echo "---"
            echo "_Generated by \`.github/workflows/develop-to-main-pr.yml\`_"
          } > pr-body.md

      - name: Open or refresh PR
        if: steps.check.outputs.ahead != '0'
        env:
          GH_TOKEN: ${{ secrets.GITHUB_TOKEN }}
        run: |
          existing=$(gh pr list --base main --head develop --state open --json number -q '.[0].number // empty')
          if [[ -n "$existing" ]]; then
            echo "Refreshing existing PR #$existing"
            gh pr edit "$existing" --body-file pr-body.md
          else
            echo "Opening new draft PR"
            gh pr create \
              --base main \
              --head develop \
              --draft \
              --title "chore(release): develop → main" \
              --body-file pr-body.md
          fi

      - name: Close PR if no commits ahead
        if: steps.check.outputs.ahead == '0'
        env:
          GH_TOKEN: ${{ secrets.GITHUB_TOKEN }}
        run: |
          existing=$(gh pr list --base main --head develop --state open --json number -q '.[0].number // empty')
          if [[ -n "$existing" ]]; then
            echo "develop has no commits ahead of main — closing stale PR #$existing"
            gh pr close "$existing" --comment "Auto-closing: \`develop\` no longer has commits ahead of \`main\`."
          else
            echo "Nothing to do — develop is up-to-date with main and no open PR exists."
          fi
```

## How it behaves

- **`develop` ahead of `main`** → opens a draft `develop → main` PR titled `chore(release): develop → main`, or refreshes an existing one's body with the current commit list.
- **`develop` level with `main`** (e.g. right after a release PR lands) → closes any stale open PR so the list stays clean.
- Uses only the built-in `GITHUB_TOKEN` — no PAT, no extra secrets.

## Notes for the report

- The PR is opened as a **draft** on purpose: it accumulates silently and only becomes a release action when the user marks it ready / merges.
- This does not replace release-please. The chain is: merge `develop → main` (this PR) → release-please opens a versioned release PR → merging that tags `vX.Y.Z` and triggers `deploy.yml`.
